# NUR-FEAT-002 需求摘录

## 关联需求

- `FR-CAT-001`（需求规格第 640-647 行）：P0，用户可按分类查看商品，分类层级建议不超过三级。
- `FR-CAT-002`（第 648-662 行）：P0，管理员可新增、编辑、停用、调整父子关系和排序、设置图片、绑定参数模板及筛选项。
- `FR-ADMIN-GOODS-001`（第 1059-1078 行）：P0，管理员维护名称、分类、编号、简介、苗圃、产地、单位、参考价、规格价、状态、推荐、排序和 SEO。
- `FR-ADMIN-GOODS-002`（第 1079-1098 行）：P0，苗木规格覆盖高度、地径、胸径、冠幅、土球、容器、等级、价格、库存、编码和图片；首期限制规格维度。
- `FR-ADMIN-GOODS-003`（第 1099-1122 行）：P0，苗木参数优先复用 ShopXO 分类参数模板。
- `FR-ADMIN-GOODS-005`（第 1137-1144 行）：P0，可上下架和重新上架，下架不得删除历史收藏、询价和统计。
- `BR-PRICE-001`（第 482-485 行）：商品上架前必须具有有效参考价格或价格区间。
- `BR-PRICE-002`（第 486-489 行）：列表和详情直接公开参考价格，不要求先询价。
- `BR-PRICE-003`（第 490-500 行）：支持固定、规格、区间和起始参考价，“面议”只允许特殊情况。
- `BR-PRICE-004`（第 501-506 行）：详情展示指定参考价格免责声明。
- `AC-001`（第 2117-2124 行）：管理员设置规格参考价并上架后，游客无需登录或询价即可在列表和详情看到与规格一致的价格。

## 任务路由

- PRIORITY: P0
- PHASE: 1
- RISK: L3

`requirement-routing.md` 将苗木目录、商品展示和公开价格列入第一阶段。价格是上架门禁且迁移会写目录、模板、台账及存量商品状态，因此按 L3 由独立产品代理批准计划和独立代码代理审查合并。

## 已解决决策

- `DEC-CATALOG-BASELINE`：首版两级、最多三级；8 个一级分类及已批准二级叶子；nursery 托管目录使用单一主归属叶子分类；SKU 最多两个维度；默认单位为株、盆、丛、平方米。
- `DEC-PRICE-VALIDITY`：规格行是唯一数值真源；严格 ASCII 十进制；草稿可为 0.00，上架每行须为 0.01..99999999.99；固定/区间由规格聚合，不新增价格模式表；列表显示最低价起，详情显示区间和规格精确价；首版禁止纯面议。
- `DEC-CATALOG-IMPORT-MODE`：以 `plugins_nursery_catalog_manifest` 台账管理 seed_key、ID、版本和结构哈希；existing 默认只增量；同父同名未托管、父级冲突或受管 ID 缺失整批失败；排序、图片、SEO、描述和启停等运营字段不被静默覆盖；fresh 清理只能进入 L4 发布合同。

## 业务目标

本任务交付“能维护、能上架、能公开看价”的目录和价格底座。新插件代码就位后，受控 CLI 可幂等建立苗木目录、规格模板和参数模板，安装/升级事件只做只读预检；商品必须选择台账内的受管二级叶子，旧演示叶子或管理员新建但尚未纳管的分类不能用于保存/上架；管理员保存草稿不受零价阻断，但任何上架路径都必须通过规格级价格、单位和目录约束；公开页面和 API 输出统一参考价语义及免责声明。

## 明确不做

- 不在本任务完成完整主题视觉、首页编排、筛选 UI、图片素材或演示商品。
- 不实现询价、收藏迁移、价格历史、统计和看板。
- 不支持纯面议或独立按商品配置的价格模式；区间商品在列表天然采用“最低价起”。
- 不删除 ShopXO 演示数据，不修改 `config/shopxo.sql`，不新增表，不修改 ShopXO 核心。
- 不安装插件、不连接服务器、不修改 Caddy；真实 PHP/MySQL/HTTP/browser 验收归后续 L4 集成发布任务。

## 开放决策

- 本任务无开放决策。`DEC-PRIO-PRICE-HISTORY` 仍为 open，但价格历史明确不在本任务范围，不阻断公开价格底座。
