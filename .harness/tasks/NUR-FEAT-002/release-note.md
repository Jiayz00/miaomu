# NUR-FEAT-002 发布与回滚说明

## 变更摘要

- 新增版本化苗木目录清单：8 个一级分类及批准的二级叶子，配套苗木规格模板、参数模板和默认库存单位。
- 商品公开价格以规格行价格为唯一数值真源；草稿允许 `0.00`，保存上架、事务提交前复核和独立上架均拒绝无效价格、超两维规格、非法单位或未纳管分类。
- 匿名商品 grid/list/slider 与详情统一显示“参考价”；区间列表显示最低价起，详情保留完整区间和规格精确价，并显示完整免责声明。
- 提供受控目录迁移及价格完整性 CLI；完整性 apply 必须绑定已审 dry-run 哈希，且不会伪造价格或删除商品历史。

## 发布前提

- 本任务完成独立合并审查并进入 `approved_for_merge`；后续生产动作必须另建 L4 operations 合同和独立 release 审批。
- L4 合同锁定目标服务器、外部 SSH 凭据引用、主机指纹、`/root/jia/miaomu` 受管根、现有 Caddy 范围及精确结构化 argv；禁止原始 ssh/scp/curl。
- 在隔离的真实 MySQL 8 兼容副本完成 PHP lint、插件安装、existing/重复迁移、并发锁、冲突回滚、保存/上架矩阵、HTTP/API 和浏览器验收。
- 生产前备份数据库和现有部署文件，记录商品、规格、分类、模板、收藏、询价及统计行数基线；确认当前没有未纳管同父同名苗木目录。
- 新代码必须先落盘，之后才能运行迁移；`BeginInstall/BeginUpgrade` 只做只读预检，不能代替 CLI migrate。

## 发布步骤

1. 由 L4 broker 只读盘点当前 ShopXO、PHP、MySQL、nursery 插件状态、容器/进程和现有 Caddy 路由，保存发布前快照。
2. 部署已批准 commit 到 `/root/jia/miaomu`，执行全部 nursery PHP 文件 `php -l`，失败立即停止且不得迁移数据。
3. 安装但暂不对外启用 nursery 插件，执行目录 `preflight --mode existing`。
4. 通过受控 argv 执行 `scripts/nursery_catalog.php migrate --mode existing --actor <actor> --run-id <unique-run-id>`；核对 JSON、台账版本、实体数量和重复执行 no-op。
5. 执行 `integrity` dry-run，完整审查返回项并保存 `items_sha256`；仅在不超过 500 项且清单获 L4 审批时，使用同一哈希执行 `integrity --apply --actor <actor> --run-id <unique-run-id> --expected-items-sha256 <sha256>`。
6. 启用 nursery 插件并重新生成/核对 `app/event.php`；确认仅 nursery 插件处于本项目允许的启用集合。
7. 保持现有 Caddy，不新增 Nginx 或第二个 Caddy；仅由 L4 合同执行已验证的必要路由调整和配置 reload。
8. 完成发布后验证并留存脱敏证据，所有门禁通过后才结束发布任务。

## 回滚触发与步骤

- 触发条件：PHP lint/迁移失败、台账或目录冲突、商品保存/上架错误、公开价格与规格不一致、PX 能力恢复、HTTP 5xx、Caddy 校验失败或关键日志异常。
- 数据迁移前失败：恢复旧代码并确认站点、后台及现有 Caddy 健康；无需数据回滚。
- 数据迁移后失败：先禁用 nursery 插件并确认生成事件不再注册其 Hook，再恢复旧代码；目录种子与台账默认保留，已下架商品不得自动重新上架。
- 需要恢复目录或商品 before-image 时，只允许使用 L4 发布前数据库备份或受审恢复清单；禁止猜测性删除受管行、商品、收藏、询价或统计。
- Caddy 变更失败时恢复发布前配置快照，先验证配置再 reload；不得引入替代网关。

## 发布后验证

- 匿名首页模块、分类、搜索、grid/list/slider 与商品详情可访问，固定价、区间价、最低价起、单位和免责声明一致；规格切换显示精确规格价。
- 后台草稿 `0.00` 可保存；零价/负价/越界/指数/全角数字、多分类、未纳管分类、超两维和非法单位的上架均失败且事务无部分写入。
- dry-run 不写数据库；apply 仅下架无效在架商品或修正派生汇总，第二次同 run-id 不重复执行；收藏、询价和统计行数保持基线。
- 购物车、订单、支付、售后及其他 PX 入口与直达路由继续不可见、不可达。
- 检查 PHP、应用、数据库和 Caddy 日志无新增高严重度错误；确认首页、搜索、详情、登录与用户中心基础流程健康。
