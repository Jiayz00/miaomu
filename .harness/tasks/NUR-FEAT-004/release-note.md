# NUR-FEAT-004 发布与回滚说明

## 变更摘要

- 登录用户可在看见公开参考价后，从商品详情或自己的可用收藏项直接发起询价；未收藏也可提交，提交不会自动收藏。
- 每条询价固定保存商品、图片、规格、提交时公开参考价、商品状态、联系信息和完整需求快照；商品后续编辑、下架、逻辑删除或缺失不改写历史。
- 用户中心新增“我的询价”；本人可查看提交内容、当前状态、管理员回复、报价有效期和只追加状态时间线。
- 后台新增询价列表、详情、回复、状态、手机号 reveal 和终态重开六项权限。列表/详情默认脱敏，完整手机号仅在专门权限下审计后返回。
- 状态机采用 `pending/replied/user_viewed/communicating/completed/closed`；回复、状态、重开和 reveal 审计只追加。回复价格是单次回复数据，不覆盖商品公开价或询价快照。
- 完全相同内容使用仓库外实例密钥 HMAC-SHA-256 做 600 秒防重；每个认证用户 60 秒最多 5 次提交尝试。新增五张 nursery 自有表和 `sxo_config` schema v1 台账。
- nursery 配置使用项目内静态 logo.svg，使非超级管理员可在 ShopXO 角色管理中看到并分配全部六项询价权限；不依赖外部图片地址。

## 发布前提

1. NUR-FEAT-004 独立 merge review 通过，发布 commit 干净、已推送且与 review-pack 一致；本 L3 任务本身不执行远程动作。
2. 后续 L4 集成发布任务锁定目标主机、PHP-FPM/MySQL/Caddy 实例、数据库名、受管目录、外部 SSH 凭据引用、备份和精确回滚动作；只复用既有 Caddy，不创建 Nginx 或第二个 Caddy。
3. 对目标数据库创建可恢复备份，记录 `sxo_config` 和五张目标表是否存在、当前询价/回复/history 行数；不得读取或导出无关个人数据。
4. PHP 8、PDO MySQL 和 `Normalizer`/intl 经真实检查；缺少 NFKC 能力时先由 L4 合同修复运行环境，否则询价写入保持失败关闭。
5. 外部实例配置提供高熵 inquiry HMAC 密钥引用，密钥不进入 Git、`sxo_config`、命令输出、日志、证据或聊天；不得为通过检查使用默认密钥。
6. nursery 插件配置中的项目内 logo 可加载，`BaseService::AdminPowerMenu()` 返回 index/detail/reply/statusupdate/contactreveal/reopen 六项；目标管理员角色按最小权限配置并刷新 ShopXO 权限缓存。
7. 停止新询价写流量后，通过项目 remote broker 运行 inquiry migration `preflight`。同名异构结构、台账冲突、备份失败或数据库权限不足必须阻止发布。

## 发布步骤

本任务不连接服务器、不执行迁移或发布。获批的后续 L4 任务应仅通过 `python -I -S -B scripts/harness.py remote-exec` 的锁定动作执行以下顺序：

1. 保留发布前代码、数据库和现有 Caddy 配置快照，关闭 nursery 询价提交入口但保持现有公开展示和收藏可读。
2. 部署已审查 commit，安装/启用运行所需的既有 PHP 扩展配置，并以外部引用注入 HMAC 密钥；不把密钥写进仓库或数据库。
3. 运行 `php scripts/nursery_inquiry.php preflight`，再以唯一 actor/run-id 执行 `migrate`；核对五表实际结构、复合唯一守卫、无外键和 `plugins_nursery_inquiry_schema_v1` 台账。
4. 运行所有修改 PHP 的 lint、五项 Python required_tests，以及真实 MySQL 中断/幂等/并发验证。任一失败停止，不开放写流量。
5. 刷新 ShopXO 插件和权限缓存，为测试角色验证六项动作权限；确认页面初始数据与日志没有完整手机号、HMAC 密钥、地址或说明泄露。
6. 执行匿名、用户 A/B、管理员角色、状态/回复、防重/限流、商品下架、公开价隔离和 PC/H5 浏览器烟雾测试。
7. 复用现有 Caddy 做站点健康检查，不新增代理服务；全部门禁通过后才重新开放询价提交并进入观察期。

## 回滚触发与步骤

触发条件包括：迁移结构不符、重复询价、限流绕过、IDOR、非法状态、孤立回复/history、完整手机号或密钥泄露、回复覆盖公开价、收藏副作用、持续 5xx/SQL 错误、PC/H5 主流程不可用或 PX 能力重新出现。

1. 立即关闭新询价提交和管理员写动作，保留用户本人及授权管理员只读历史；不得通过删表或删数据止损。
2. 保存脱敏错误、受影响 ID、时间窗口和日志引用，禁止复制完整手机号、地址、用户说明、回复正文、canonical payload 或 HMAC 密钥。
3. 回退应用代码到发布前 commit，并恢复发布前 PHP-FPM/Caddy 配置；继续复用既有 Caddy。若问题仅为 schema 未完成，保持写入口关闭并用相同 schema v1 清单前向修复。
4. 五张询价表、所有快照、回复、history、reveal 审计、防重/限流行和 `sxo_config` 台账全部保留，不执行 `DROP/TRUNCATE/DELETE`。只有另立 L4 数据任务且证明无业务记录、无依赖、备份可恢复时才允许考虑结构删除。
5. 核对回滚前后询价/回复/history/审计行数不减少，商品与规格公开价、收藏行数不变化；运行五项自动测试和只读 HTTP 烟雾后再决定是否恢复写入。

## 发布后验证

- 首页、搜索、苗木列表和详情继续公开显示参考价及免责声明；收藏与立即询价并列，购物车、订单、支付、供应商和多商户入口仍不可见不可达。
- 匿名用户被引导登录；用户 A 未收藏可提交且不自动收藏，用户 B 篡改 ID 无法访问 A 的询价，错误不泄露存在性。
- 快照与提交时公开价在商品改价、改规格、下架和逻辑删除后保持不变；历史详情仍可读，当前商品不可用时链接禁用。
- 600 秒完全重复在串行/并发下只产生一条，拒绝不刷新窗口；不同需求可提交。每用户 60 秒第 6 次尝试拒绝，其他用户独立计数。
- 管理回复、用户首次查看、沟通、完成/关闭和授权重开符合状态矩阵；回复与 history 数量对应，无孤立行，回复报价未修改商品或快照价格。
- 后台筛选和 24 小时未首次回复超时规则正确；普通页面只含脱敏手机号，contactreveal 未授权拒绝、授权动作先有审计再返回且日志保持脱敏。
- nursery 项目内 logo 正常显示，非超级管理员可以按角色分配 index/detail/reply/statusupdate/contactreveal/reopen，未分配动作直达拒绝。
- PHP-FPM、MySQL 和既有 Caddy 日志在观察期无持续错误、死锁风暴、个人数据或密钥泄露；PC/H5 表单、列表、时间线、弹层和窄屏布局无重叠。
