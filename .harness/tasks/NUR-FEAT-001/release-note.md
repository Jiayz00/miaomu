# NUR-FEAT-001 发布与回滚说明

## 变更摘要

新增项目级 `nursery` 插件。启用后，ShopXO 前台、API、后台和已知商城插件入口将收敛为单主体苗木展示站：移除购物车、结算、订单、支付、售后、积分等入口和直达路由，并清理后台首页遗留快捷入口，保留分类、搜索、商品公开价格、登录、资料、收藏和浏览历史。分类与搜索图文列表、默认主题动态商品 slider 使用插件同构视图，不再输出硬编码购物车图标；显式 default 回退同样收敛，非 default 主题自有模板不被覆盖。8 个尚未获得首版授权的默认插件只隐藏入口、不永久扩大直达拒绝；发布时只允许 nursery 插件启用。用户中心首页只在外层空 view 或显式 default 回退时改用插件视图，内部 header/nav/user-menu/footer 等模块继续正常渲染；本任务不新增询价入口、数据库表或完整苗木视觉主题。

## 发布前提

- 本任务只批准源码合并，不批准远程安装或生产发布。
- 发布必须由独立 L4 operations 任务锁定服务器、Git commit、外部 SSH 凭据引用、现有 Caddy 边界、备份和精确动作，并取得独立 release approval 与 release seal。
- 运行环境需具备与 ShopXO 6.9.0 匹配的 PHP/Composer/MySQL；发布前对应用文件、数据库和现有 Caddy 配置做受控快照。
- 必须先在测试环境执行 PHP 语法检查、插件发现/安装/启用、`app/event.php` 生成、路由矩阵、权限、副作用和页面测试，并确认除 nursery 外没有插件被启用。用户中心请求必须设置超时与输出上限，证明外层模板只渲染一次且嵌套公共模块不递归。任一正向路由误伤、负向路由可达、递归渲染、内存错误或 500 均阻塞发布。

## 发布步骤

本任务不执行发布。后续 L4 任务应通过项目 remote broker 部署已批准 commit，使用 ShopXO 标准插件安装与启用流程生成事件映射，刷新应用缓存；不得手工编辑 `app/event.php`，不得新建 Nginx 或第二个 Caddy。

## 回滚触发与步骤

触发条件：插件安装/启用失败、PHP 错误、允许路由被误拒、排除路由仍可达、后台权限未收敛、数据库出现意外写入或页面关键入口异常。

回滚顺序：在维护窗口禁用 nursery 插件，刷新事件映射与缓存并确认 `app/event.php` 不再引用 nursery；确认分类、搜索和动态商品模块恢复部署前的默认模板行为后，再回退到发布前 commit。恢复发布前 Caddy/应用快照仅能由 L4 回滚动作执行。不得卸载或物理删除 ShopXO 商城核心表和历史数据。

## 发布后验证

1. 确认首页、分类、搜索、商品列表/详情、登录、资料、安全、收藏和浏览历史正常；分类图文列表、搜索 `layout=1` 和动态商品 slider 无购物车图标，同时公开价格、单位和详情链接仍可用。
2. 在 default 主题、非 default 主题自有用户页和非 default 主题回退 default 三种路径访问用户中心，确认外层 nursery 用户模板只渲染一次，header/nav/header-top/user-menu/footer 各正常出现且 PHP/ShopXO 日志无递归、内存或超时错误。
3. 对 Web 8 个、API 10 个、后台 12 个控制器及 23 个 PX 插件逐项执行正负路由矩阵，覆盖混合大小写和非 `index` action。
4. 检查 PC/H5 导航、用户中心、商品按钮、后台菜单/权限在冷缓存和暖缓存下均无排除能力。
5. 对购物车、订单、支付、售后、用户、收藏和商品表做发布前后计数及更新时间抽样，确认拒绝请求无业务副作用。
6. 检查 PHP、ShopXO 与 Caddy 错误日志；本任务不含询价和统计功能，不把其缺失记为本任务回归失败。
