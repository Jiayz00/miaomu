# NUR-FEAT-001 发布与回滚说明

## 变更摘要

新增项目级 `nursery` 插件。启用后，ShopXO 前台、API、后台和已知商城插件入口将收敛为单主体苗木展示站：移除购物车、结算、订单、支付、售后、积分等入口和直达路由，并清理后台首页遗留快捷入口，保留分类、搜索、商品、登录、资料、收藏和浏览历史。8 个尚未获得首版授权的默认插件只隐藏入口、不永久扩大直达拒绝；发布时只允许 nursery 插件启用。用户中心首页改用插件视图；本任务不新增询价入口、数据库表或苗木视觉主题。

## 发布前提

- 本任务只批准源码合并，不批准远程安装或生产发布。
- 发布必须由独立 L4 operations 任务锁定服务器、Git commit、外部 SSH 凭据引用、现有 Caddy 边界、备份和精确动作，并取得独立 release approval 与 release seal。
- 运行环境需具备与 ShopXO 6.9.0 匹配的 PHP/Composer/MySQL；发布前对应用文件、数据库和现有 Caddy 配置做受控快照。
- 必须先在测试环境执行 PHP 语法检查、插件发现/安装/启用、`app/event.php` 生成、路由矩阵、权限、副作用和页面测试，并确认除 nursery 外没有插件被启用。任一正向路由误伤或负向路由可达均阻塞发布。

## 发布步骤

本任务不执行发布。后续 L4 任务应通过项目 remote broker 部署已批准 commit，使用 ShopXO 标准插件安装与启用流程生成事件映射，刷新应用缓存；不得手工编辑 `app/event.php`，不得新建 Nginx 或第二个 Caddy。

## 回滚触发与步骤

触发条件：插件安装/启用失败、PHP 错误、允许路由被误拒、排除路由仍可达、后台权限未收敛、数据库出现意外写入或页面关键入口异常。

回滚顺序：在维护窗口禁用 nursery 插件，刷新事件映射与缓存并确认 `app/event.php` 不再引用 nursery；再回退到发布前 commit。恢复发布前 Caddy/应用快照仅能由 L4 回滚动作执行。不得卸载或物理删除 ShopXO 商城核心表和历史数据。

## 发布后验证

1. 确认首页、分类、搜索、商品列表/详情、登录、资料、安全、收藏和浏览历史正常。
2. 对 Web 8 个、API 10 个、后台 12 个控制器及 23 个 PX 插件逐项执行正负路由矩阵，覆盖混合大小写和非 `index` action。
3. 检查 PC/H5 导航、用户中心、商品按钮、后台菜单/权限在冷缓存和暖缓存下均无排除能力。
4. 对购物车、订单、支付、售后、用户、收藏和商品表做发布前后计数及更新时间抽样，确认拒绝请求无业务副作用。
5. 检查 PHP、ShopXO 与 Caddy 错误日志；本任务不含询价和统计功能，不把其缺失记为本任务回归失败。
