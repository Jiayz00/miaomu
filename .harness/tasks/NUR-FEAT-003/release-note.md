# NUR-FEAT-003 发布与回滚说明

## 变更摘要

- 登录用户可在苗木卡片、详情和基础“我的收藏”页显式收藏或取消收藏；重复请求保持目标状态，不会反向切换。
- 收藏按认证用户隔离；商品下架、逻辑删除或缺失后记录保留并显示不可用状态。
- 新增 `(user_id, goods_id)` 版本化唯一索引迁移，历史重复时失败关闭且不自动删除数据。
- 旧 toggle、旧用户收藏写入口和商品物理删除入口被拒绝；收藏不创建或修改询价。

## 发布前提

1. 独立合并审查通过，发布提交干净且可追溯。
2. L4 集成任务锁定目标实例、数据库备份、PHP/MySQL 版本和回滚动作。
3. 在停止收藏写流量后运行 `php scripts/nursery_favorite.php preflight`；重复数据、同名错误索引或异常台账必须阻止发布。
4. 对 `sxo_goods_favor` 与 `sxo_config` 做可恢复备份并记录收藏总行数、重复组数和询价/事件表基线。
5. nursery 插件已启用，其他 PX 插件保持未启用；现有 Caddy 由既有网关管理，不创建 Nginx 或第二个 Caddy。

## 发布步骤

本任务不执行远程发布。后续 L4 任务应通过项目 remote broker：部署已审查提交，运行 favorite `preflight`，再以唯一 actor/run-id 执行 `migrate`；确认真实唯一索引和台账后才开放收藏写路由。随后执行 PHP lint、HTTP/API、A/B 用户、CSRF、下架保留、旧入口拒绝和 PC/H5 浏览器烟雾测试。

## 回滚触发与步骤

触发条件：迁移失败、收藏重复、越权、旧写路由可达、页面不可用或错误日志持续增长。先停止收藏写流量，再回退应用代码到前一发布提交；保留唯一索引和迁移台账，不删除或合并收藏数据。恢复前核对收藏总行数、每用户商品唯一性、询价/事件表未变化，并保持商品物理删除入口关闭。

## 发布后验证

- 首页、搜索、列表和详情公开参考价继续显示，购物车/订单/支付入口仍不可见不可达。
- 匿名点击收藏进入登录；用户 A/B 的 add、重复 add、cancel、重复 cancel、status/list 均按用户隔离。
- 下架、逻辑删除和缺失商品仍保留收藏行并正确标记，所属用户可取消。
- 旧 Web/API 收藏写 URL 与 `admin/goods/delete` 返回拒绝；商品上下架仍可用。
- 收藏前后询价和事件表行数不变；PHP-FPM、MySQL、Caddy 日志无新增持续错误。
- PC 与 H5 收藏按钮、列表空态、分页和窄屏布局无重叠。
