# 需求追踪与开放决策

## 追踪要求

每个业务任务至少关联一个可在需求文档中定位的 `BR-*`、`FR-*`、`NFR-*`、`DATA-*`、`METRIC-*` 或 `AC-*` 编号，并声明优先级、阶段、业务不变量、修改路径、数据影响、测试、回滚和证据。

## 阶段顺序

1. 基础改造与商品展示：版本、菜单、复用基线、分类参数、公开价格；主要验收 `AC-001`。
2. 用户与收藏：注册、用户中心、收藏和下架状态；主要验收 `AC-002`、`AC-005`。
3. 询价：快照、用户/后台流程、回复历史、权限和防重复；主要验收 `AC-003~005`、`AC-009`。
4. 流量数据：访客、会话、事件目录、搜索/商品/收藏/询价采集和日汇总；主要验收 `AC-006~008`。
5. 数据看板：总览、分析、排行与经批准的导出；要求原始数据与汇总对账。
6. 增强：价格历史、收藏价格变化、留存渠道、通知告警；主要验收 `AC-010`。

## 自动门禁重点

- 收藏唯一性、用户归属和 IDOR 越权测试。
- 询价快照、回复/状态日志和历史保留。
- 迁移、回滚、幂等、唯一约束和关键索引。
- 事件名白名单、必填字段、敏感字段扫描以及 PV/UV/指标公式对账。
- PX 功能的入口、路由和权限不可达，而不是删除上游源码。
- 实际测试命令、退出码、截图或数据库断言。
